# Карта Памяти — Образовательное приложение об исторических землях Ингушетии
## Пошаговый план реализации

---

## Что за приложение

Интерактивная образовательная платформа, объединяющая:
- **Исторические карты** Кавказского края (XVIII–XXI вв.) с наложением административных границ разных эпох
- **Временную шкалу** ключевых событий: депортации, присоединения к России, советские автономии, 2018 г.
- **Энциклопедию** обществ и племён (галгай, карабулаки, цоринцы, джерахи, аккинцы и др.)
- **Архивные документы** — оцифрованные первоисточники, письма, рапорты
- **Этимологический словарь** топонимов (Мереджа, Фортанга, Алкун, Цечахки и т.д.)
- **ДНК-атлас** гаплогрупп по обществам (J2, L3, J1)

Главная цель — дать любому человеку доступ к академическому уровню знаний об Ингушетии через красивый, простой интерфейс.

## Стек технологий

- **Spring Boot 4 / Java 25** — REST API

- **Leaflet.js** — интерактивные карты с кастомными слоями
- **React Router** — маршрутизация между разделами

### Мобильный клиент (позже)
- **PWA** (Progressive Web App) — сначала, без нового кода
- **React Native** — если нужно нативное приложение (использует тот же API)

## Ответ на вопрос про мобильные приложения

**Spring Boot бэкенд НЕ создаёт проблем для Android/iOS.**

Схема работы:
```
Android App   ──┐
iOS App       ──┤──► Spring Boot API (Fly.io) ──► PostgreSQL
Web (React)   ──┘         REST/JSON
```

Spring Boot — это просто HTTP-сервер, который возвращает JSON. Ему всё равно, кто обращается: браузер, телефон или Postman. Для мобильного приложения нужно только написать отдельный фронтенд на React Native или Flutter, который будет делать запросы к уже задеплоенному API. Бэкенд переписывать не нужно.

Если не нужны push-уведомления и нативные фичи телефона — достаточно PWA: одна кодовая база, работает в браузере на телефоне "как приложение".

### Этап 0 — Инфраструктура (1–2 дня)
**Цель:** поднять новый Spring Boot проект, БД, деплой.

1. Создать новый Spring Boot проект (`IngushHistoryApp`)
2. Подключить PostgreSQL:
   - Локально: Docker Compose (`postgres:16`)
   - На Fly.io: `fly postgres create`
3. Создать базовую структуру таблиц (миграции через Flyway):
   ```sql
   CREATE TABLE events (id, year, title_ru, description_ru, category, lat, lon, source_ref);
   CREATE TABLE societies (id, name_ru, name_ing, description_ru, territory_geojson, era_from, era_to);
   CREATE TABLE documents (id, title, year, author, text_ru, archive_ref, image_url);
   CREATE TABLE toponyms (id, name_ru, name_ing, etymology_ru, modern_name, lat, lon);
   CREATE TABLE persons (id, name_ru, years, role_ru, biography_ru);
   ```
4. Настроить CORS для React-фронта
5. Создать репозиторий на GitHub (github.com/Morgxth/ingush-history-app)
6. Настроить Dockerfile + fly.toml (по аналогии с SomeApp)

- 1934 — объединение в ЧИ АССР
- 1944 — депортация, передача Пригородного района СО АССР

- 1991 — референдум (97.4% за Ингушскую Республику)
- 1992 — Закон «Об образовании Ингушской Республики»
- 1992 — осетино-ингушский конфликт, Пригородный район
- 2018 — Соглашение о границе с Чечнёй, протесты

### Этап 4 — Интерактивная карта (3–4 дня)
**Цель:** главная фича приложения.

#### Технология
- **Leaflet.js** + React-обёртка `react-leaflet`
- Базовый тайловый слой: OpenStreetMap или Stadia Maps (бесплатно)

#### Слои карт
1. **Современные границы** (GeoJSON, доступен OpenStreetMap)
2. **Исторические административные единицы** — нарисовать полигоны вручную по документу:
   - Ингушевский округ 1864 г.
   - Владикавказский округ 1869 г.
   - Ингушская АО 1924 г.
   - ЧИ АССР 1934 г.
   - Республика Ингушетия с 1992 г.
   - Спорный участок Фортанги до/после 26.09.2018
3. **Сёла** — маркеры с попапами (ныне существующие и исчезнувшие)
4. **Этнические границы** — из Этнографической карты АН СССР 1928–1930 гг.

#### Переключатель эпох
Слайдер по годам (1800–2025): слои автоматически меняются, показывая как менялись границы.

#### Маркеры событий
При включении слоя "События" на карте появляются иконки с ключевыми датами.
